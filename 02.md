# CORD-02: Chatrooms

Chatrooms are comprised of private channels that combine to display the chat history. The channels are split across two *planes*:

1. **Chat plane**: user chat messages. The channel key changes over time.
2. **Control plane**: priority chatroom state. The channel key typically remains the same.

## Invites

Users are invited to chatrooms using an invite link. The invite link contains a shared secret.

```
https://vectorapp.io/invite/#hDHlmtklzYrsjxtaQ-HwgNy0YxLmMT5IOp7MRkVgjvg
```

The initial secret is derived from the chatroom creator's pubkey, allowing participants to verify the chatroom creator:

```js
shared_secret = sha256("concord/room" + owner_pubkey + owner_salt) // TODO: owner_salt goes in the URL?
```

The shared secret is used to derive the channel keys for both planes.

## Planes

Planes are groups of channels defined by their key derivation from the shared secret.

### Chat Plane

The chat plane contains usual user chat activity. Clients can load the newest data first and then paginate by querying older events. It uses a rotating channel key depending on the current time.

```js
epoch = math.floor(now() / 1_000_000) // ~12 days
chat_key = hkdf(shared_secret, "concord" + epoch)
```

Querying the chat plane works like querying any other private channel(s):

```json
{ "kinds": [1059], "authors": ["<...channel pubkeys>"] }
```

### Control Plane

The control plane contains critical state events that clients must sync.

```js
control_key = hkdf(shared_secret, "concord/control")
```

Clients should aim to keep the entire control plane in sync, while downloading data from the chat plane as needed.
